## <a name="permissions"></a><span data-ttu-id="781b7-133">권한</span><span class="sxs-lookup"><span data-stu-id="781b7-133">Permissions</span></span>

<span data-ttu-id="781b7-134">WSL의 권한과 관련하여 다음 두 가지 중요한 개념을 명심해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="781b7-134">There are two important concepts to keep in mind when it comes to permissions in WSL:</span></span>

<span data-ttu-id="781b7-137">WSL에서 Linux를 실행하는 경우 Linux는 이를 시작하는 프로세스와 동일한 Windows 권한을 갖습니다.</span><span class="sxs-lookup"><span data-stu-id="781b7-137">When running Linux on WSL, Linux will have the same Windows permissions as the process that launches it.</span></span> <span data-ttu-id="781b7-138">Linux는 다음 두 가지 권한 수준 중 하나로 시작할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="781b7-138">Linux can be launched in one of two permission levels:</span></span>

* <span data-ttu-id="781b7-140">관리자 권한/관리자: Linux가 관리자 권한/관리자 Windows 권한으로 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="781b7-140">Elevated/admin: Linux runs with elevated/admin Windows permissions</span></span>

<span data-ttu-id="781b7-142">위의 Windows 권한은 Linux 인스턴스 내의 권한과는 독립적입니다.
